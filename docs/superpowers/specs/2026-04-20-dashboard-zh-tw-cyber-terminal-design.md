# Dashboard 繁體中文與資安終端風格改版設計

日期：2026-04-20

## 目標

將現有 dashboard 介面改為：

- 全面使用繁體中文介面文案
- 視覺風格改成高密度、偏 SOC 監控台的資安終端介面
- 保留既有資料來源與 API，不改 backend 行為
- 強化即時監控感與掃讀效率，而不是只做換色皮膚

本次改版聚焦於前端靜態資產：

- `public/index.html`
- `public/app.js`
- `public/styles.css`

## 使用者需求摘要

使用者希望：

- 介面改成繁體中文
- 具備高科技動畫感
- 整體風格偏資安終端，而不是一般 SaaS 卡片介面
- 版面採用「上方狀態列 + 下方雙欄主工作區」
- 使用感偏真實監控台，而不是展示型 landing page

## 視覺論點

本次介面採用「深色資安指揮台」方向：

- 基底為石墨黑與藍黑色階
- 以螢光青作為主操作與高亮色
- 輔以少量綠色或琥珀色表達系統狀態
- 背景使用低干擾格線、漸層霧層與掃描式光感
- 避免厚重卡片牆與傳統企業後台白底排版

設計原則：

- 可讀性優先於裝飾
- 高科技感來自構圖、光感、節奏與狀態回饋，不靠大量特效
- 動畫必須幫助辨識新事件、選取狀態與連線狀態

## 版面架構

### 1. 頂部狀態列

頂部區域改為監控台 header，而不是傳統網站 topbar。

內容包含：

- 產品名稱與副標
- 全域搜尋欄
- 系統狀態 chips

狀態 chips 至少顯示：

- Proxy 連接埠
- CA 憑證狀態
- 即時串流狀態
- 目前 session 數量

### 2. 主工作區

主工作區改為雙欄配置：

- 左欄：流量事件
- 右欄：封包剖析 + 節點診斷

#### 左欄：流量事件

負責顯示即時 session feed。

每筆事件至少顯示：

- HTTP method
- host + path
- scheme
- response status 或 pending 狀態
- 時間

視覺上採 event feed 風格，不做傳統白底按鈕清單。

#### 右欄上半：封包剖析

負責顯示目前選取事件的詳細資料。

內容包含：

- 事件摘要
- request headers
- request body preview
- response headers
- response body preview

資料區塊會用高密度框格與終端風格 `pre` 呈現。

#### 右欄下半：節點診斷

負責顯示：

- proxy port
- 憑證狀態
- CA 下載入口
- Android 設定步驟

此區定位為控制面板與操作提示區，不只是單純說明文字。

## 文案策略

所有使用者可見文案改為繁體中文，並避免生硬直譯。

建議核心文案：

- `Android Proxy Dashboard` -> `Android 代理監控台`
- `Android Traffic Capture` -> `Android 流量擷取`
- `Session List` -> `流量事件`
- `Session Detail` -> `封包剖析`
- `Setup & Diagnostics` -> `節點診斷`
- `Filter sessions` -> `檢索流量`

placeholder、錯誤訊息、空狀態與載入提示也需一併繁中化。

## 互動與動畫

### 1. 載入進場

頁面初始載入時：

- 狀態列與主區塊有短暫的淡入與上浮
- 背景格線與掃描效果維持低頻存在

進場時間需短，不可拖慢工具感。

### 2. 新事件提示

當 SSE 收到新 session：

- 新增的事件項目有一次短脈衝高亮
- 不做整欄重排動畫，不製造閃爍干擾

### 3. 選取狀態

被選中的 session 需有明確且穩定的視覺識別：

- 外框高亮
- 側邊能量條
- 背景光帶

右側詳細區標題應同步反映目前選取狀態。

### 4. 系統狀態回饋

以下狀態需視覺化：

- SSE 已連線
- SSE 重連中
- 憑證存在
- 憑證尚未建立

做法以狀態點、色彩與細微 pulse 為主，不使用浮誇動畫。

## 元件與樣式策略

### HTML

`public/index.html` 需重組：

- 將現有 topbar 改為監控台 header
- 將三欄 panel 改成頂部狀態列 + 雙欄主區
- 保留既有主要掛載點或以最小範圍調整 selector

### CSS

`public/styles.css` 需重寫為主，但仍維持純 CSS 實作。

需要定義：

- 新色彩變數
- 背景格線與光感層
- 高密度 panel 邊界與內發光效果
- 狀態 chip 樣式
- session feed 樣式
- 右側 detail / diagnostics 模組化區塊
- 響應式行為

### JavaScript

`public/app.js` 不改資料存取模型，但需調整渲染文案與局部結構。

需補強：

- 即時串流狀態呈現
- 中文 placeholder / 錯誤提示
- setup 區塊繁中文案
- 新 session 的短暫 highlight 標記

## 響應式行為

桌面版為主要使用情境。

但仍需確保：

- 中尺寸螢幕下，主工作區可合理堆疊
- 手機寬度下不破版
- 搜尋欄、狀態 chip 與 session 列表仍可操作

行動版不追求完整桌面體驗，但必須可讀可用。

## 錯誤處理與空狀態

下列情境需有一致的繁中呈現：

- setup 載入失敗
- sessions 載入失敗
- 單筆 session 詳情載入失敗
- 尚無任何流量
- 篩選後無符合結果
- 尚未選取任何事件

空狀態文案應符合監控台語氣，但仍保持清楚直接。

## 測試與驗證

本次變更以 UI 靜態資產為主，驗證方式如下：

- `npm test`
- `tsc --noEmit`
- 手動打開 dashboard 驗證版面與文字
- 確認 `/api/setup`、`/api/sessions`、`/api/events` 既有流程不受影響

手動驗證重點：

- 首屏是否為繁體中文
- 深色終端風格是否成立
- 新 session 進來時是否有短暫高亮
- 選取 session 是否清楚
- setup 區塊是否仍能操作下載 CA
- 窄螢幕下是否合理堆疊

## 非目標

本次不做：

- backend API 調整
- 新增圖表或統計模組
- 多頁路由
- WebSocket 取代 SSE
- 新增國際化框架
- 重新設計資料結構

## 實作完成標準

當以下條件成立，視為本次改版完成：

- 所有主要 UI 文案改為繁體中文
- 版面改為頂部狀態列 + 下方雙欄主工作區
- 視覺風格明確轉為資安終端 / SOC 指揮台
- 至少具備三種有感但克制的動態回饋
- 現有 session、detail、setup 流程維持可用
- 測試與型別檢查通過
